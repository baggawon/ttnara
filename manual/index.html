테더나라 (TTNARA) 관리자 운영 매뉴얼

이 매뉴얼은 어드민 패널의 모든 페이지가 무엇을 하고, 어떻게 사용하며, 어떤 주의사항이 있는지를 비개발자도 이해할 수 있도록 설명합니다.

1대시보드

메뉴

대시보드 /admin/dashboard

🎯목적

플랫폼 운영 현황(사용자 추이, 거래량, 평점 분포, 시스템 상태, 상위 트레이더, 신고 내역, 토픽 활동 등)을 실시간으로 한눈에 보는 종합 모니터링 화면입니다.

✅주요 기능

  • KPI 카드(사용자 수, 거래 수, 활성 사용자, 평점 통계) 조회
  • 신규 사용자 가입 추이 그래프
  • 완료 거래량 추이 그래프
  • 상위 트레이더 리더보드
  • 사용자 평점 분포 차트
  • 시스템 상태(가동 시간, API 응답, DB 상태) 모니터링
  • 최근 사용자 신고 내역
  • 토픽별 활동 수준
  • 현재 플랫폼 설정 요약 (레벨/사용자/시스템)

📋화면 구성

  • KPI 행: 사용자 수 / 거래 수 / 활성 사용자 / 평점 통계
  • 사용자 추이 카드: 가입자 시계열 그래프
  • 거래 추이 카드: 거래 완료 시계열 그래프
  • 상위 트레이더 카드: 거래 횟수 기준 리더보드
  • 평점 분포 카드: 사용자 평점 히스토그램
  • 최근 신고 카드: 신고자/대상자 정보
  • 토픽 활동 카드: 토픽별 참여 횟수
  • 시스템 상태 카드: 가동 시간 / API 응답 / DB 상태

⚠️주의사항

이 화면은 읽기 전용입니다. 데이터 수정은 불가능하며, 차트와 통계는 실시간(자동 새로고침) 기준입니다.

메뉴

설정 /admin/general

🎯목적

플랫폼의 전역 설정(일반·시스템·레벨·사용자)을 한 화면에서 관리합니다. 네 개의 독립 카드(폼)로 구성되며, 각 카드는 독립적으로 저장합니다.

✅주요 기능

  • 일반 설정: 사이트명, 설명 등 기본 정보
  • 시스템 설정: 환경, 점검 모드 등 시스템 동작
  • 레벨 설정: 최대 시스템 레벨, 포인트 계산
  • 사용자 설정: 닉네임 길이 제한, 이메일 검증 등 회원 가입 관련

📋화면 구성

  1. 일반 설정 카드
  2. 시스템 설정 카드
  3. 레벨 설정 카드
  4. 사용자 설정 카드

⚠️주의사항

전역 설정이므로 변경 사항은 즉시 모든 사용자·모든 게시판에 적용됩니다. 일부 설정은 캐시 갱신이 필요할 수 있습니다.

회원 관리 · 상세

회원 상세 (읽기 전용) /admin/users/[user_id]

🎯목적

한 회원의 프로필·계정 상태·KYC·거래 통계를 한 화면에서 읽기 전용으로 조회합니다.

✅주요 기능

  • 기본 정보 조회 (사용자명, 이메일, 닉네임, 가입일)
  • 프로필 통계 조회 (레벨, 포인트, 등급, 거래 횟수, 권한 레벨, 상태)
  • 권한 상태 조회 (관리자, 활성화, 보증 여부)
  • KYC 정보 조회 (인증 결과, 금융기관, 계좌번호, 예금주, 등록일)
  • 수정 페이지로 이동
  • 목록으로 돌아가기

📋화면 구성

  • 기본 정보 카드: 사용자명 / 닉네임 / 이메일 / 가입일
  • 통계 카드: 사용자 레벨 / 거래 등급 / 포인트 / 거래 횟수 / 권한 레벨 / 계정 상태(녹색·빨강 점)
  • 권한 카드: 관리자 권한 / 활성화 상태 / 거래 보증 상태 토글 (읽기 전용)
  • KYC 카드: 인증 결과 / 금융기관 / 계좌번호 / 예금주 / 등록일

⚠️주의사항

모든 토글은 표시용이며 수정되지 않습니다. KYC가 "미등록"인 경우 KYC 카드 세부 정보는 표시되지 않습니다.

등급 (목록) /admin/ranks

🎯목적

사용자 거래 등급(랭크)을 관리합니다. 단건 생성/수정/삭제, 자동 생성, 일괄 편집, 배지 이미지 관리 진입점을 제공합니다.

✅주요 기능

  • 이름/설명으로 등급 검색 (검색 필드 선택 가능)
  • 정렬 (최신/오래된)
  • 단건 등급 생성
  • 자동 등급 생성 페이지로 이동
  • 일괄 편집
  • 배지 이미지 관리로 이동
  • 단건 삭제 · 등급 수정 페이지로 이동

📋화면 구성 (테이블 컬럼)

이름 / 등급 레벨 / 최소 거래 횟수 / 배지 이미지 / 활성화 / 작업 버튼

⚠️주의사항

  • 등급 레벨은 유일해야 합니다 (중복 불가, 보통 1·2·3 순차).
  • 상위 등급의 최소 거래 횟수는 하위 등급보다 커야 합니다.
  • 배지 이미지는 여러 등급이 공유할 수 있어, 해제 시 모든 사용 등급에 영향.
  • 등급 삭제 시 해당 등급 보유 사용자는 한 단계 낮은 등급으로 떨어집니다.
자동 생성은 기존 등급을 모두 삭제한 뒤 재생성합니다. (파괴적 작업)

등급 · 자동 생성

자동 등급 생성 파괴적 /admin/ranks/auto

🎯목적

최대 등급 수·최대 거래 횟수·진행 곡선을 지정해 전체 등급을 자동 생성합니다. 시뮬레이션 후 저장합니다.

✅주요 기능

  • 최대 등급 수 / 최대 거래 횟수 입력
  • 진행 유형 선택 (선형 / 오목 / 볼록)
  • 진행 속도 조정 (1–5, 비선형일 때만)
  • 계산 버튼으로 시뮬레이션
  • 결과 그리드 + 차트 확인
  • 저장 버튼으로 확정 (확인 다이얼로그 후)

📋화면 구성

  • 입력 필드: 최대 등급 수 / 최대 거래 횟수 / 진행 유형 / 진행 속도
  • 결과 그리드: 등급별 카드 (등급 9개 초과 시 처음 3 + 중간 3 + 마지막 3개만 표시)
  • 결과 차트: X=등급 레벨, Y=필요 거래 횟수

⚠️주의사항

저장 시 기존 등급을 전부 삭제하고 새로 생성합니다. 되돌릴 수 없습니다.
진행 유형 가이드:
  • 선형: 균등 증가
  • 오목: 초반 완만, 후반 가파름 (신규 사용자 친화적)
  • 볼록: 초반 가파름, 후반 완만

등급 · 배지 관리

배지 이미지 관리 /admin/ranks/badges

🎯목적

등급 배지 이미지를 업로드하고, 특정 등급 범위에 일괄 할당·해제합니다.

✅주요 기능

  • 배지 이미지 업로드 (이름 선택 입력)
  • 등급 범위(예: 1~5)로 일괄 할당
  • 범위 수정 / 할당 해제
  • 배지 이미지 삭제
  • 전체 배지 목록 보기

📋화면 구성

  • 업로드 섹션: 이름 입력 + 파일 선택

거래 시스템 제어 파괴적 /admin/system-control

🎯목적

P2P 거래 시스템 비상 제어 도구. 3개의 파괴적 작업을 확인 다이얼로그·확인 문구와 함께 제공합니다.

✅주요 기능

1) 거래 게시 일시 중단 (P2P Pause)
  • ON: 신규 거래 글 작성 차단
  • OFF: 정상 운영
  • 진행 중 거래·제안은 영향 없음
  • 토글 시 확인 다이얼로그 표시
2) 모든 거래 취소 (Cancel All Active Trades)
  • 모든 진행/공개 거래·제안을 일괄 취소
  • 완료된 거래는 보존
  • 취소된 거래에 대한 trade_count 감소
  • 확인 문구 "모든 거래 취소" 입력 필요
3) 거래 기록 리셋 (Full Reset)
  • 모든 tether / tether_proposal / tether_rate 삭제
  • 모든 사용자의 trade_total, trade_count, trade_joined, trade_rate 초기화
  • 모든 사용자의 등급을 최저 등급으로 초기화
  • 모든 리더보드(전체·주간·일간) 초기화
  • 확인 문구 "거래 기록 리셋" 입력 필요

📋화면 구성

  • Pause 카드: 상태 아이콘 (PauseCircle/PlayCircle) + 상태 텍스트 (일시 중단/정상 운영) + 토글
  • Cancel All 카드: 경고 배너 + 설명 + 빨간색 파괴 버튼 + 확인 다이얼로그
  • Full Reset 카드: 위험 배너 + 삭제·초기화 항목 목록 + 빨간색 파괴 버튼 + 확인 다이얼로그

⚠️주의사항 (Destructive — 절대 주의)

다음을 반드시 인지하고 사용하세요:
  • Pause: 즉시 모든 사용자에게 영향.
  • Cancel All: 되돌릴 수 없음. 완료 거래는 보존되지만 진행 거래는 모두 종료.
  • Full Reset: 수개월~수년의 거래 이력을 영구 삭제. 데이터 복구 불가.
  • 확인 문구를 정확히 입력해야 실행됩니다 — 실수 방지 장치.
  • 비상시·점검 시간·서비스 종료 시에만 사용하세요.
  • 실행 시 외부에 별도 로그를 남겨 감사 추적을 유지하세요.

    • 삭제된 지역 복구
    • "삭제된 지역 표시" 체크박스 토글

⚠️주의사항

  • 이름은 validateTetherCategoryName 검증 통과 필요.
  • 변경은 즉시 API로 저장됩니다 (전통적 폼 제출 없음).

부록 D — 관리자(Admin) vs 권한 레벨(Auth Level), 그리고 모더레이션

이 앱에는 권한을 결정하는 독립된 두 가지 시스템이 동시에 존재하며, 경계가 직관적이지 않아 자주 혼동됩니다. 이 부록은 그 관계를 명확히 정리합니다.

1. 두 가지 권한 시스템

시스템 DB 필드 UI 라벨 역할
어드민 권한 (이진값) is_app_admin (Boolean)
또는 is_superuser		 관리자 권한 /admin/* 백오피스 접근을 결정하는 유일한 게이트. ON이면 어드민 전체 사용 가능, OFF면 접근 불가.
권한 레벨 (숫자) auth_level (Int, 기본 1) 권한 레벨 / 시스템레벨 사용자 화면(게시판·다운로드·작성·모더레이션)에서의 행동 가능 범위. 게시판마다 요구 레벨이 다름.
핵심: 두 시스템은 독립적으로 작동합니다. 어드민이라고 자동으로 권한 레벨이 올라가지 않으며, 권한 레벨이 높다고 어드민이 되지 않습니다.

2. 무엇이 무엇을 가로막는가

동작 필요 조건 비고
/admin/* 백오피스 접근 is_app_admin = ON (단독) auth_level은 무관. 미들웨어가 이 한 가지 조건만 검사.
특정 게시판 글 읽기 is_app_admin=ON 또는 auth_level >= 토픽.level_read 둘 중 하나만 만족하면 됨.
글 작성 / 댓글 / 다운로드 is_app_admin=ON 또는 auth_level >= 토픽.level_create/comment/download 동일 패턴.
게시판 모더레이션 (남의 글 수정/삭제 등) is_app_admin=ON 또는 auth_level >= 토픽.level_moderator 토픽별 "주제 관리권한 레벨" 설정값에 따라 모더레이터가 결정됨.
채팅 모더레이션 (뮤트/차단/숨김) is_app_admin = ON (단독) auth_level 경로 없음. 어드민만 가능.

3. "모더레이터"라는 별도 역할은 없습니다

이 앱에는 is_moderator 같은 별도 필드가 존재하지 않습니다. 모더레이션 권한은 다음 두 가지 경로 중 하나로만 부여됩니다:

  • 전역 모더레이터: is_app_admin=ON — 모든 게시판·채팅에서 모더레이션 가능
  • 특정 게시판 모더레이터: auth_level >= 해당 토픽의 level_moderator — 그 게시판에서만 모더레이션 가능

따라서 "이 게시판만 관리할 수 있는 모더레이터"를 만들고 싶다면:

  1. 해당 토픽의 level_moderator를 적절한 숫자(예: 50)로 설정 (/admin/boards/topics/[topic])
  2. 대상 사용자의 auth_level을 그 숫자 이상으로 올려줌 (/admin/users/[id]/edit)
  3. 관리자 권한은 OFF로 유지 — 어드민 백오피스에는 들어올 수 없게 됨

4. 자주 묻는 시나리오

상황결과
관리자 권한 OFF, 권한 레벨 99 ❌ 어드민 백오피스 진입 불가. ✅ 모든 게시판에서 작성·다운로드·(레벨이 맞는 토픽에서) 모더레이션 가능.
관리자 권한 ON, 권한 레벨 1 ✅ 어드민 백오피스 전체 사용 가능. 사용자 화면에서는 평범한 회원처럼 보이지만, 모더레이션 동작에 대해서는 어드민이라는 이유로 통과됨.
관리자 권한 ON, 권한 레벨 99 ✅ 모든 것 가능 — 일반적인 운영자 셋업.
특정 게시판만 관리하는 외부 운영자 관리자 권한 OFF + 권한 레벨을 해당 토픽 level_moderator 이상으로. 백오피스에는 못 들어오지만 그 게시판은 모더레이션 가능.

5. 운영상 주의사항

권한 승급의 위험성:
  • 어드민은 다른 사람을 어드민으로 승급시킬 수 있습니다. 회원 수정 화면(/admin/users/[id]/edit)에서 "관리자 권한" 토글을 ON으로 저장하면 됩니다 — 추가 검증(예: 본인의 권한 레벨이 충분히 높은지)은 없습니다. 따라서 어드민 권한은 신중히 부여하세요.
  • 관리자 권한을 끈다고 권한 레벨이 함께 떨어지지 않습니다. 둘은 독립이므로 어드민에서 해제할 때는 권한 레벨도 함께 조정해야 사용자 화면에서의 권한도 회수됩니다.
  • 채팅 모더레이션 위임 불가: 채팅의 뮤트·차단·숨김·신고 처리는 권한 레벨로 위임할 수 없습니다. 채팅 운영을 맡기려면 반드시 그 사람에게 어드민 권한을 부여해야 합니다.
참고: src/helpers/config.ts의 admins = ["admin", "superAdmin"] 상수는 UI 표시용 라벨일 뿐이며 권한 인증과 무관합니다. 실제 권한은 is_app_admin / is_superuser DB 필드만 결정합니다.

6. 한눈에 보는 의사결정 가이드

원하는 것해야 할 일
운영팀 전체 권한 부여 관리자 권한 ON + 권한 레벨을 사용 환경에 맞게 (보통 최상위)
특정 게시판만 관리하는 보조 모더레이터 관리자 권한 OFF + 권한 레벨 ≥ 해당 토픽의 level_moderator
비공개 게시판 출입 권한 권한 레벨 ≥ 해당 토픽의 level_read (관리자 권한 불필요)
채팅 운영 위임 관리자 권한 ON 외에는 방법 없음
외부 협력자에게 일부 게시판만 노출 해당 토픽의 level_* 값을 낮추고, 그 사용자의 권한 레벨만 약간 올림
